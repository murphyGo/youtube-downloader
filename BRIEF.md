# youtube-downloader

> One page. Prose with bullets.

## What this is

A YouTube video downloader with two faces: a static web UI on GitHub Pages for paste-and-download, and a CLI for agent automation. Both share the same download logic via a small Fly.io proxy.

## The problem

I want to download YouTube videos (regular videos and Shorts) without depending on sketchy third-party sites. I also want an agent-callable CLI for automated workflows.

## MVP — what v1 must do

- Accept a YouTube URL (full `watch?v=...` and `/shorts/...` formats) and resolve a download
- Web UI (GH Pages): paste URL → click button → file downloads
- CLI: `ytdl <url>` downloads to the current directory

## Tech

- **Language**: Python — `yt-dlp` is the de-facto tool for this; using it for both the CLI and the proxy keeps the stack unified
- **Framework**: `yt-dlp` for extraction, FastAPI + Uvicorn for the Fly.io proxy
- **UI**: Static HTML + vanilla JS on GH Pages (no build step in v1)
- **Data**: none — stateless, no persistence
- **Deploy**: GH Pages (UI) + Fly.io (proxy, streams files back) + local `pip install` (CLI)

## Out of scope

Explicitly NOT in v1:
- Authentication, accounts, per-user rate limiting
- Playlists (single-video only)
- Subtitle / caption extraction
- Format / quality selector (use a sensible default — best mp4)
- Live streams
- Mobile app

## Done means

I can paste a YouTube video or Shorts URL into the GH Pages site and the file downloads. I can also run `ytdl <url>` from the terminal and get the same file. Agents can shell out to the CLI.
